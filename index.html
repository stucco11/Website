    Personal

    • PixelSort

      This is a project of mine that I worked on over Winter of 2020. I’ve always been interested by pixel-sorting, so I made an application that allows users to have access to a number of different options to sort user-specified images with. These options include:

      • Brightness - Evaluates the brightness of each pixel and sorts in order from 0 to 1 (The user is able to manually set the upper and lower bounds for filtering out specific brightness values)
      • Hue - Goes in the order of Red, Orange, Yellow, Green, Blue, Violet
      • Saturation - Evaluates the saturation of each pixel and sorts in order from 0 to 1
      • RGB - Sorts pixels based on the respective amounts of Red, Green, or Blue in the row (whether the program sorts by Reds, Greens, or Blues is specified by the user)

      Additional options exist that work in tandem with the selected sorting method:

      • Extending - this sorts the image as though it was a single line of pixels
      • Spiral Sort - this recursively sorts the enclosing perimeter of the image as though it was a singular line before stopping at the middle
      • Partitioning (Vertical and Horizontal) - this will partition the image by a number of user defined lines. This allows the program to sort each partition as though it were an individual image before stitching everything back together

      Some future goals that I have for this project are:

      • Include rotation
      • Seed-based sorting

      Language(s) Used: C#, XAML, WPF

    • Game Shelf

      This is a personal project that I used to teach myself C#, WPF, and XAML while becoming familiar with Visual Studio and the MVVM pattern. I also used it to become familiar with Azure Dev Ops. I eventually want to create a web-based version as well.

      Game Shelf is a program that allows the user to search for the title of a game, pick the correct game from a returned list, and add the game to a collection (such as Playing, Want to Play, Dropped, etc.). This is done by using GiantBomb.com's public API to request information about the specified search term and displaying the returned results to the user in a condescend manner. The associated information and images of the selected game are then downloaded to a folder and act as a cache.

      Some future goals that I have for this project are:

      • Rewrite this program to be based on the web using HTML5 and JavaScript
      • Allow the user to have more options for sorting games in their collection, including by genre, publisher, console, release year, ect.
      • Allow the user to track how many hours they have spent playing each individual game
      • Incorporate data from HowLongToBeat.com to be used alongside how much time the user has spent playing each game
      • Utilize Steam's API to allow the user to import their games from their Steam libraries, then interface with GiantBomb to download the appropriate data for each of these titles
      • Create a banner/ribbon that appears over games that are completed
      • Achievement tracking
      • Interface overhall
      • Light/dark themes
      • Eventually find a GiantBomb successor

      Language(s) Used: C#, XAML, WPF
